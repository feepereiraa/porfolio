Featured Projects

Standardization of Data Environment with Cloud-Based ELT Pipeline

With the goal of standardizing and documenting General Shopping’s data environment, I was responsible for implementing an ELT pipeline. The project used a PostgreSQL database, DBT (Data Build Tool) for data transformations, and Apache Airflow for batch orchestration. The entire infrastructure was deployed in Oracle Cloud Infrastructure, designed to run 24/7.

Customer Base Unification Project

Using Python, SQL, and Apache Airflow, I developed an ETL pipeline that integrates eight different customer data sources from General Shopping. The pipeline performs deduplication and data enrichment, preparing the unified dataset for strategic use by the company’s CRM. Out of approximately 5 million records, the final deduplicated base contained around 2.3 million unique individuals. This enabled the identification of consumption patterns, persona creation, and a stronger customer relationship strategy.

Cloud Infrastructure Management on OCI

I was responsible for setting up the company’s foundational cloud infrastructure, including compute instances, storage buckets, databases, and user accounts. I also configured the virtual network (VPC), subnets, firewall rules (ACLs), and routing policies, in addition to managing access control across services. This work ensured a secure, organized, and scalable environment for cloud-based data projects.

CI/CD Automation for Data Projects

I implemented a CI/CD framework to automate the deployment of dbt models into production, using GitHub Actions with a self-hosted runner on a VM within Oracle Cloud. This pipeline standardized the validation, testing, and deployment process, improving both agility and reliability. The solution also empowered the data team with greater control over versioning and the deployment flow of new tables.

Centralization and Standardization of SAP Data

To centralize and standardize data from SAP, I defined the strategic reports to be extracted, modeled their structure into MySQL tables, and configured the continuous data ingestion process. This initiative ensured greater consistency, accessibility, and organization of information for both analytical and operational use..
